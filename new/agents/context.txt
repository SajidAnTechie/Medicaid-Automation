================================================================================
  SENTINEL-STATE — Medicaid Fee Schedule Automation Pipeline
  Project Context & Agent Reference
================================================================================

OVERVIEW
--------
Sentinel-State is an automated pipeline that discovers, downloads, extracts,
normalises, and stages Medicaid fee-schedule data from U.S. state portals.

The pipeline is composed of four autonomous agents, each deployed as a
standalone container on AWS AgentCore. Every agent exposes a single
`POST /invocations` endpoint and communicates via JSON payloads. The output
of one agent feeds directly into the input of the next.

TECH STACK
----------
  • Framework       : strands-agents SDK (Agent, @tool, BedrockModel)
  • LLM             : Amazon Bedrock (Claude models)
  • Runtime         : AWS AgentCore (BedrockAgentCoreApp, @app.entrypoint)
  • Container Base  : ghcr.io/astral-sh/uv:python3.13-bookworm-slim
  • Storage         : Amazon S3 (raw → medicaid-fee-raw, staged → medicaid-fee-stage)
  • Region          : eu-north-1 (AgentCore & S3), us-east-1 / eu-north-1 (Bedrock)

================================================================================
  AGENT 1 — NAVIGATOR  ("The Researcher")
================================================================================
  Location  : agents/navigator/
  Entrypoint: agents.navigator.agentcore_navigator
  Key Files :
    • agent.py              – System prompt, create_navigator_agent(), invoke()
    • models.py             – NavigatorInput (portal_url), NavigatorOutput
    • tools/crawl_portal.py – Playwright BFS crawl (AgentCore Browser service
                              in prod, local Chromium in dev)
    • tools/filter_datasets.py – Union-Find dedup, format prioritisation,
                                  LLM relevance scoring

    Receives a state Medicaid portal URL. Crawls the site using a headless
    browser, discovers all downloadable fee-schedule files, deduplicates them,
    and returns a ranked list of relevant datasets.

      "prompt": "<state_portal_url>"
    }

  AGENT 2 — EXTRACTOR  ("The Parser")
================================================================================
  Location  : agents/extractor/
  Entrypoint: agents.extractor.agentcore_extractor
  Key Files :
    • agent.py                  – run_extractor(), save_extracted_data_to_csv()
    • models.py                 – ExtractorInput, ExtractorOutput, ExtractedTable
    • agentcore_extractor.py    – @app.entrypoint, payload unwrapping

  PURPOSE:
    Receives the Navigator output (specifically the relevant_datasets list).
    Downloads the top dataset file, detects sheets/tables, extracts all rows,
    maps raw column headers to canonical names using LLM analysis, and returns
    structured data with quality metrics.

  INPUT:
    {
      "prompt": "<JSON string of the full Navigator response>"

================================================================================
  AGENT 3 — CSV EXPORTER
================================================================================
  Location  : agents/csv_exporter/
  Entrypoint: agents.csv_exporter.raw_csv_exporter
  Key Files :
    • raw_csv_exporter.py       – @app.entrypoint mainFunction(), run_csv_exporter()
    • models.py                 – CSVExporterInput, CSVExporterOutput
    • tools/download.py         – download_or_cache_file()
    • tools/reader.py           – read_and_map_data()
    • tools/writer.py           – export_to_csv()

  PURPOSE:
    Receives the Extractor output. Downloads the original file (or uses a
    cached copy), reads the specified sheet, applies column mappings, and
    exports the raw data as a CSV file. Uploads the CSV to S3 (raw bucket).

    When the payload contains "extracted_tables" (Extractor output shape),
    the entrypoint auto-maps the nested structure to flat CSVExporterInput
    fields (sheet_name, column_mapping, output_path, etc.).

  INPUT:
    {
      "prompt": "<JSON string of the full Extractor response>"
    }

  OUTPUT:
    {
      "success": true,
      "rows_exported": 9552,
      "columns": ["Proc Code", "Proc Mod", "OMK", …],
      "output_path": "s3://medicaid-fee-raw/raw_exports/alaska/alaska_raw_data.csv",
      "error": null,
      "metadata": {
        "source_url": "https://…/Fee_Schedule_Physician_FY2026.xlsx",
        "state_name": "Alaska",
        "category": "physician",
        "cached_file_used": false
      }
    }

================================================================================
  AGENT 4 — ANALYSIS  ("The Mapper")
================================================================================
  Location  : agents/analysis/
  Entrypoint: agents.analysis.agentcore_analysis
  Key Files :
    • agentcore_analysis.py     – @app.entrypoint invoke(), run_analysis(),
                                  run(), create_analysis_agent(), chunking logic

  PURPOSE:
    Receives the S3 path of the raw CSV from the CSV Exporter. Fetches the CSV,
    analyses headers using an LLM, creates canonical schema-compliant mappings,
    cleans and transforms all data (unicode cleanup, rate→float, flag→bool),
    exports the cleaned CSV, and uploads it to S3 (stage bucket).

  INPUT:
    {
      "prompt": "<S3 URI from csv_exporter output_path>"
    }

    Example:
    {
      "prompt": "s3://medicaid-fee-raw/raw_exports/alaska/alaska_raw_data.csv"
    }

  OUTPUT:
    {
      "success": true,
      "output_filepath": "s3://medicaid-fee-stage/cleaned_data/alaska/alaska_raw_data_cleaned.csv"
    }

  CANONICAL SCHEMA FIELDS:
    procedure_code  (str)   – HCPCS or CPT code, e.g. "D0120" or "99213"
    modifier        (str)   – optional procedure modifier, e.g. "TC" (null if absent)
    fee_amount      (float) – reimbursement rate as a plain number, e.g. 45.50
    effective_date  (str)   – ISO date YYYY-MM-DD (null if not present)
    end_date        (str)   – ISO date YYYY-MM-DD (null if not present)

  END-TO-END EXAMPLE  (Alaska Physician Fee Schedule)
================================================================================

  Step 1 — Navigator
    Input : {"prompt": "https://extranet-sp.dhss.alaska.gov/.../FeeSchedule.html"}
    Output: Discovers 24 links, returns ranked list with

  Step 2 — Extractor
    Input : {"prompt": "<Navigator JSON>"}
    Output: Downloads xlsx, extracts "Physician Fee Schedule" sheet,
            9551 rows, 9 columns, mapping_confidence 0.9

  Step 3 — CSV Exporter
    Input : {"prompt": "<Extractor JSON>"}
    Output: Exports 9552 rows to
            s3://medicaid-fee-raw/raw_exports/alaska/alaska_raw_data.csv

  Step 4 — Analysis
    Input : {"prompt": "s3://medicaid-fee-raw/raw_exports/alaska/alaska_raw_data.csv"}
    Output: Cleaned & canonical CSV uploaded to
            s3://medicaid-fee-stage/cleaned_data/alaska/alaska_raw_data_cleaned.csv

================================================================================
  DEPLOYMENT NOTES
================================================================================
  • Each agent has its own Dockerfile under .bedrock_agentcore/<agent_name>/
  • Build context is the agent directory (e.g. agents/navigator/)
  • Package structure is preserved: COPY . /app/agents/<agent_name>/
  • A top-level /app/agents/__init__.py is created in the Dockerfile
  • Containers run as non-root user "bedrock_agentcore" (uid 1000)
  • Temp files written to /tmp/ (container filesystem is ephemeral)
  • Deploy with: bedrock-agentcore deploy --no-cache
  • Payload from AgentCore may be multi-layer JSON-encoded; all entrypoints
    use loop-based unwrapping to handle arbitrary nesting depth
